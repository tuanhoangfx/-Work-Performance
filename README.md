# Infi Project - Task Manager

## üìñ Project Overview

Infi Project Task Manager is a modern, elegant internal tool designed for managing team member tasks through an intuitive, Kanban-style interface. It focuses on a smooth user experience, clear data presentation, and a dedicated dashboard for managers.

## ‚ú® Key Features

*   **Kanban Board**: An interactive board for team members to view and manage their tasks, organized by status ('To Do', 'In Progress', 'Done').
*   **Task Management**: Users can create, update, and delete their own tasks. Each task can have a title, description, status, priority, and due date.
*   **Task Comments**: A full commenting system on each task to facilitate team communication.
*   **File Attachments**: Attach files (images, documents) to tasks for better context and information sharing, with support for direct pasting from the clipboard.
*   **Real-time Notifications**: Get notified instantly when you're assigned a task or when someone comments on your tasks.
*   **Priority and Status**: Clearly defined priorities ('Low', 'Medium', 'High') and statuses help in organizing and tracking work efficiently.
*   **Comprehensive Manager Dashboard**:
    *   A separate view for users with an 'admin' role.
    *   View a list of all team members and select any team member to view their task board.
    *   Full CRUD (Create, Read, Update, Delete) functionality for tasks for any team member.
    *   Includes an 'Overall Summary' view showing aggregated statistics for all tasks in the system.
*   **Secure Authentication & Authorization**: Powered by Supabase Auth with Row Level Security (RLS) to ensure users can only access their own data, while managers have full access.
*   **Customizable UI**: Switch between Light/Dark modes with beautiful gradient color themes.
*   **Multi-language Support**: English, Vietnamese (Ti·∫øng Vi·ªát), and Thai (‡πÑ‡∏ó‡∏¢).
*   **Responsive Design**: A fully responsive layout for all screen sizes.
*   **In-App User Guide**: A comprehensive guide available directly within the application.

## üíª Tech Stack

-   **Frontend**: React, TypeScript, Tailwind CSS
-   **Backend**: Supabase (Database, Auth, Storage, RLS, Realtime)
-   **State Management**: React Context API & Component State

## üõ†Ô∏è Setup Instructions

To get the application running with your own Supabase backend, follow these steps.

### 1. Create a Supabase Project

1.  Go to [supabase.com](https://supabase.com/) and sign in.
2.  Create a new project. Save your **Project URL** and **anon key**.

### 2. Database Schema Setup

Go to the **SQL Editor** in your Supabase project dashboard and run the following script. This single script will set up everything you need for a new database.

```sql
-- This script is designed to be run on a fresh Supabase project.
-- It sets up all necessary tables, types, functions, triggers, and security policies.

-- 1. CUSTOM ENUM TYPES
-- These ensure data consistency for task status, priority, and user roles.
CREATE TYPE public.task_status AS ENUM ('todo', 'inprogress', 'done', 'cancelled');
CREATE TYPE public.user_role AS ENUM ('employee', 'admin');
CREATE TYPE public.task_priority AS ENUM ('low', 'medium', 'high');


-- 2. TABLE CREATION
-- PROFILES Table: Stores public user information linked to Supabase's auth.users.
CREATE TABLE public.profiles (
  id uuid NOT NULL PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  updated_at timestamptz,
  full_name text,
  avatar_url text,
  role public.user_role NOT NULL DEFAULT 'employee'
);
COMMENT ON TABLE public.profiles IS 'Stores public profile information for users.';

-- TASKS Table: The main table for storing all tasks.
CREATE TABLE public.tasks (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
  created_by uuid REFERENCES public.profiles(id) ON DELETE SET NULL,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now(),
  title text NOT NULL,
  description text,
  status public.task_status NOT NULL DEFAULT 'todo',
  priority public.task_priority NOT NULL DEFAULT 'medium',
  due_date date
);
COMMENT ON TABLE public.tasks IS 'Stores tasks for users.';
CREATE INDEX ON public.tasks (user_id);
CREATE INDEX ON public.tasks (status);

-- TASK_ATTACHMENTS Table: Stores file attachments for tasks.
CREATE TABLE public.task_attachments (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  task_id bigint NOT NULL REFERENCES public.tasks(id) ON DELETE CASCADE,
  user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
  file_name text NOT NULL,
  file_path text NOT NULL,
  file_type text,
  file_size bigint,
  created_at timestamptz NOT NULL DEFAULT now()
);
COMMENT ON TABLE public.task_attachments IS 'Stores file attachments for tasks.';
CREATE INDEX ON public.task_attachments (task_id);

-- TASK_TIME_LOGS Table: Stores time tracking entries for each task.
CREATE TABLE public.task_time_logs (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    task_id bigint NOT NULL REFERENCES public.tasks(id) ON DELETE CASCADE,
    user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
    start_time timestamptz NOT NULL,
    end_time timestamptz,
    created_at timestamptz NOT NULL DEFAULT now()
);
COMMENT ON TABLE public.task_time_logs IS 'Stores work time logs for tasks.';
CREATE INDEX ON public.task_time_logs (task_id);
CREATE INDEX ON public.task_time_logs (user_id);

-- ACTIVITY_LOGS Table: Stores records of user actions within the app.
CREATE TABLE public.activity_logs (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamptz NOT NULL DEFAULT now(),
  user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
  task_id bigint REFERENCES public.tasks(id) ON DELETE SET NULL,
  action text NOT NULL,
  details jsonb
);
COMMENT ON TABLE public.activity_logs IS 'Records user activities within the application.';
CREATE INDEX ON public.activity_logs (user_id);
CREATE INDEX ON public.activity_logs (task_id);

-- TASK_COMMENTS Table: Stores comments for tasks.
CREATE TABLE public.task_comments (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamptz NOT NULL DEFAULT now(),
  task_id bigint NOT NULL REFERENCES public.tasks(id) ON DELETE CASCADE,
  user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
  content text NOT NULL CHECK (content <> '')
);
COMMENT ON TABLE public.task_comments IS 'Stores comments for tasks.';
CREATE INDEX ON public.task_comments (task_id);
CREATE INDEX ON public.task_comments (user_id);

-- NOTIFICATIONS Table: Stores notifications for users about important events.
CREATE TABLE public.notifications (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamptz NOT NULL DEFAULT now(),
  user_id uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE, -- Recipient
  actor_id uuid REFERENCES public.profiles(id) ON DELETE SET NULL, -- User who performed the action
  type text NOT NULL, -- e.g., 'new_task_assigned', 'new_comment'
  data jsonb, -- Relevant data like task_id, task_title
  is_read boolean NOT NULL DEFAULT false
);
COMMENT ON TABLE public.notifications IS 'Stores notifications for users.';


-- 3. HELPER FUNCTIONS AND TRIGGERS
-- Automatically updates the `updated_at` column whenever a task is modified.
CREATE OR REPLACE FUNCTION public.moddatetime()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = now();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER handle_updated_at BEFORE UPDATE ON public.tasks
  FOR EACH ROW EXECUTE PROCEDURE public.moddatetime();

-- Automatically creates a profile in the `profiles` table when a new user signs up.
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER
LANGUAGE plpgsql
SECURITY DEFINER SET search_path = public
AS $$
BEGIN
  INSERT INTO public.profiles (id, full_name, avatar_url, role)
  VALUES (new.id, new.raw_user_meta_data->>'full_name', new.raw_user_meta_data->>'avatar_url', 'employee');
  RETURN new;
END;
$$;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
  
-- Helper function to check if the current user is an admin.
CREATE OR REPLACE FUNCTION public.is_admin()
RETURNS boolean
LANGUAGE sql
SECURITY DEFINER
AS $$
  SELECT EXISTS (
    SELECT 1
    FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  );
$$;

-- Notification function for new task assignments.
CREATE OR REPLACE FUNCTION public.create_new_task_assignment_notification()
RETURNS TRIGGER
LANGUAGE plpgsql
SECURITY DEFINER SET search_path = public
AS $$
DECLARE
  assigner_name text;
BEGIN
  SELECT full_name INTO assigner_name FROM profiles WHERE id = auth.uid();
  IF NEW.user_id IS NOT NULL AND NEW.user_id <> auth.uid() THEN
    INSERT INTO public.notifications (user_id, actor_id, type, data)
    VALUES (
      NEW.user_id,
      auth.uid(),
      'new_task_assigned',
      jsonb_build_object(
        'task_id', NEW.id,
        'task_title', NEW.title,
        'assigner_name', assigner_name
      )
    );
  END IF;
  RETURN NEW;
END;
$$;

CREATE TRIGGER on_new_task_created_for_other
  AFTER INSERT ON public.tasks
  FOR EACH ROW
  EXECUTE PROCEDURE public.create_new_task_assignment_notification();

CREATE TRIGGER on_task_reassigned
  AFTER UPDATE OF user_id ON public.tasks
  FOR EACH ROW
  WHEN (OLD.user_id IS DISTINCT FROM NEW.user_id)
  EXECUTE PROCEDURE public.create_new_task_assignment_notification();

-- Notification function for new comments.
CREATE OR REPLACE FUNCTION public.create_new_comment_notification()
RETURNS TRIGGER
LANGUAGE plpgsql
SECURITY DEFINER SET search_path = public
AS $$
DECLARE
  task_info record;
  commenter_name text;
BEGIN
  SELECT user_id, created_by, title INTO task_info FROM tasks WHERE id = NEW.task_id;
  SELECT full_name INTO commenter_name FROM profiles WHERE id = NEW.user_id;

  IF task_info.user_id IS NOT NULL AND task_info.user_id <> NEW.user_id THEN
    INSERT INTO public.notifications (user_id, actor_id, type, data)
    VALUES (
      task_info.user_id,
      NEW.user_id,
      'new_comment',
      jsonb_build_object('task_id', NEW.task_id, 'task_title', task_info.title, 'commenter_name', commenter_name)
    );
  END IF;

  IF task_info.created_by IS NOT NULL AND task_info.created_by <> NEW.user_id AND task_info.created_by <> task_info.user_id THEN
    INSERT INTO public.notifications (user_id, actor_id, type, data)
    VALUES (
      task_info.created_by,
      NEW.user_id,
      'new_comment',
      jsonb_build_object('task_id', NEW.task_id, 'task_title', task_info.title, 'commenter_name', commenter_name)
    );
  END IF;
  RETURN NEW;
END;
$$;

CREATE TRIGGER on_new_comment
  AFTER INSERT ON public.task_comments
  FOR EACH ROW
  EXECUTE PROCEDURE public.create_new_comment_notification();


-- 4. ROW LEVEL SECURITY (RLS)
-- Enable RLS for all tables.
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.tasks ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.task_attachments ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.task_time_logs ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.activity_logs ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.task_comments ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.notifications ENABLE ROW LEVEL SECURITY;

-- Policies for `profiles` table
CREATE POLICY "Authenticated users can view all profiles." ON public.profiles FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "Users can update their own profile." ON public.profiles FOR UPDATE USING (auth.uid() = id);
CREATE POLICY "Users can insert their own profile." ON public.profiles FOR INSERT WITH CHECK (auth.uid() = id);
CREATE POLICY "Admins can manage all profiles." ON public.profiles FOR ALL USING (public.is_admin());

-- Policies for `tasks` table
CREATE POLICY "Users can create tasks." ON public.tasks FOR INSERT WITH CHECK (auth.uid() = created_by);
CREATE POLICY "Users can view, update, delete tasks they are assigned to or created." ON public.tasks FOR ALL USING (auth.uid() = user_id OR auth.uid() = created_by);
CREATE POLICY "Admins can manage all tasks." ON public.tasks FOR ALL USING (public.is_admin());

-- Policies for `task_attachments` table
CREATE POLICY "Users can manage attachments for their tasks." ON public.task_attachments FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Admins can manage all attachments." ON public.task_attachments FOR ALL USING (public.is_admin());

-- Policies for `task_time_logs` table
CREATE POLICY "Users can manage their own time logs." ON public.task_time_logs FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Admins can manage all time logs." ON public.task_time_logs FOR ALL USING (public.is_admin());

-- Policies for `activity_logs` table
CREATE POLICY "Authenticated users can view all logs." ON public.activity_logs FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "Users can create logs for their own actions." ON public.activity_logs FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Admins can manage all logs." ON public.activity_logs FOR ALL USING (public.is_admin());

-- Policies for `task_comments` table
CREATE POLICY "Authenticated users can view all comments." ON public.task_comments FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "Users can add comments." ON public.task_comments FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can update their own comments." ON public.task_comments FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "Users can delete their own comments." ON public.task_comments FOR DELETE USING (auth.uid() = user_id);
CREATE POLICY "Admins can manage all comments." ON public.task_comments FOR ALL USING (public.is_admin());

-- Policies for `notifications` table
CREATE POLICY "Users can view their own notifications." ON public.notifications FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can update their own notifications." ON public.notifications FOR UPDATE USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Admins can manage all notifications." ON public.notifications FOR ALL USING (public.is_admin());


-- 5. STORAGE BUCKETS AND POLICIES
-- Setup Storage for avatars and attachments.
INSERT INTO storage.buckets (id, name, public) VALUES ('avatars', 'avatars', true) ON CONFLICT (id) DO NOTHING;
INSERT INTO storage.buckets (id, name, public) VALUES ('task-attachments', 'task-attachments', true) ON CONFLICT (id) DO NOTHING;

-- Policies for 'avatars' bucket
CREATE POLICY "Avatars are publicly accessible." ON storage.objects FOR SELECT USING (bucket_id = 'avatars');
CREATE POLICY "Users can manage their own avatar." ON storage.objects FOR ALL USING (bucket_id = 'avatars' AND auth.uid()::text = (storage.foldername(name))[1]) WITH CHECK (bucket_id = 'avatars' AND auth.uid()::text = (storage.foldername(name))[1]);
CREATE POLICY "Admins can manage all avatars." ON storage.objects FOR ALL USING (public.is_admin()) WITH CHECK (public.is_admin());

-- Policies for 'task-attachments' bucket
CREATE POLICY "Users can manage their own attachments." ON storage.objects FOR ALL USING (bucket_id = 'task-attachments' AND auth.uid()::text = (storage.foldername(name))[1]) WITH CHECK (bucket_id = 'task-attachments' AND auth.uid()::text = (storage.foldername(name))[1]);
CREATE POLICY "Admins can manage all attachments." ON storage.objects FOR ALL USING (bucket_id = 'task-attachments' AND public.is_admin()) WITH CHECK (bucket_id = 'task-attachments' AND public.is_admin());

```

### 3. Storage Setup

1.  Go to the **Storage** section in your Supabase dashboard.
2.  Create a new bucket named `avatars`. **Make it a Public bucket.**
3.  Create another new bucket named `task-attachments`. **Make it a Public bucket.**

*Note: The SQL script in step 2 attempts to create these buckets, but it's good practice to ensure they exist via the UI.*

### 4. Connect Your Project

Finally, open the `lib/supabase.ts` file in the editor and replace the placeholder credentials with your own **Project URL** and **anon key**.
